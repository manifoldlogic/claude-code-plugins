▸ PLUGIN STRUCTURE
.claude-plugin/plugin.json Metadata registry: name, version, hooks, permissions
agents/ Markdown agent definitions with system prompts and capability declarations
commands/ Slash commands (e.g., /epic, /sdd-gate) with YAML frontmatter and implementation
hooks/ Lifecycle hooks: SessionStart, PreToolUse, PostToolUse, Stop, PreCompact, SessionEnd
skills/ SKILL.md + supporting scripts and templates for reusable workflows

PreToolUse: Input validation before tool execution. exit 2 blocks the operation.
Stop: Workflow guidance at task boundaries. exit 2 prevents workflow continuation.
SessionStart/End: Symmetric setup/teardown for isolation and resource management.
PreCompact: Optimization hook before context compaction in long sessions.

▸ SAFETY & INTELLIGENCE ARCHITECTURE

Safety Modules

  • block-dangerous-git.py — Prevents destructive VCS operations
  • block-catastrophic-commands.py — Blocks unsafe shell operations
  • workflow-guidance.py — Enforces task completion gates
  • warn-sdd-refs.py — SDD reference validation

Agent Model Strategy

  • Haiku: Mechanical tasks (file I/O, tool orchestration, data formatting)
  • Sonnet: Reasoning and analysis (code review, system design, debugging)
  • Opus: Complex decisions (architecture choices, cross-domain synthesis, novel problem solving)
